Appendix B: Manually Writing Shellcode for Windows

Shellcode describes a type of program, which only exists to be loaded into memory and then executed. Traditionally, hackers used shellcode to create shell sessions on a target machine, but now it is common for shellcode programs to do much more specific tasks, like Credential Dumping and performing Lateral Movement. It is important to note, that unlike regular executables, shellcode does not inherently contains sections and there are no instructions for managing a language runtime. Shellcode instructions are written in machine code, so they are interpreted directly by the CPU. Shellcode is typically executed in one of two ways: either by injecting it into a vulnerable service or by being intentionally loaded into memory by a malicious program.

Writing shellcode for Unix/Linux systems is different from writing it on Windows systems. On linux, writing shellcode can be broken into 3 basic parts:

  1. Specify the system call number
  2. Put the system call parameters on the stack/in registers
  3. Use syscall or int 0x80 to execute the system call

This is repeated as needed to execute different functions. On windows, system call numbers are not made public and even if they are discovered they ay be changed across versions of Windows. Instead, when writing shelcode, we use Win32 API calls to achieve our objectives. This process works as follows:

  1. Obtain the kernel32.dll base address.
  2. Find the address of the GetProcAddress function.
  3. Use GetProcAddress to find the address of the LoadLibrary function.
  4. Use LoadLibrary to load a DLL (such as user32.dll) into memory.
  5. Use GetProcAddress to find the address of a function (such as MessageBox) from within the loaded library.
  6. Specify the function parameters.
  7. Call the function.
